# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с кодом в данном репозитории.

> **Правило самообновления:** Этот файл и файлы в `.claude/rules/` должны обновляться при изменении связанных частей приложения (архитектура, зависимости, соглашения, команды).

## Команды сборки и разработки

```bash
# Кодогенерация (модели, DI, маршруты, i18n, темы, ассеты)
make generate

# Очистка артефактов сборки и всех сгенерированных файлов
make clean

# Запуск приложения
flutter run

# Запуск тестов
flutter test
flutter test test/path/to_specific_test.dart

# Анализ кода
flutter analyze

# Форматирование кода (ширина строки: 100)
dart format lib test
```

## Кодогенерация

Проект активно использует build_runner. После изменения любого аннотированного класса выполните `make generate`.

Шаблоны сгенерированных файлов (исключены из линтинга, редактировать вручную запрещено):
- `*.freezed.dart` — иммутабельные модели Freezed
- `*.g.dart` — JSON-сериализация, клиенты Retrofit, маршруты GoRouter, базы данных Drift, ThemeTailor, ассеты flutter_gen
- `*.config.dart` — конфигурация DI через Injectable

## Архитектура

Подробное описание архитектуры см. в `.claude/rules/architecture.md`.

**Слои:** Presentation (BLoC + UI) -> Data (Repositories) -> Services (API / Local DB)

**Основные паттерны:**
- **BLoC** для управления состоянием (flutter_bloc)
- **Freezed** для иммутабельных моделей и union-типов
- **GetIt + Injectable** для внедрения зависимостей
- **Retrofit + Dio** для API-клиентов
- **Drift** для локального кэширования в SQLite
- **GoRouter** с типизированными маршрутами для навигации
- **ThemeTailor** для расширений темы
- **Slang** для интернационализации (en, ru)

## Структура проекта

```
lib/
  app/          # Инфраструктура: DI, роутер, тема, БД, окружение, i18n, сгенерированные ассеты
  data/         # Модели (Freezed), репозитории, сервисы (API + локальные)
  presentation/ # Экраны и виджеты фич, каждый с BLoC-контроллером
  shared/       # Общие расширения и виджеты для всех фич
```

Фичи следуют шаблону:
```
presentation/<feature>/screens/<screen>/
  index.dart          # Основной виджет экрана
  body.dart, header.dart, ...  # Составные части
  controller/         # bloc.dart, event.dart, state.dart
```

## Стиль кода

Полные соглашения см. в `.claude/rules/code_style.md`.

Основные правила:
- Одинарные кавычки, trailing commas, ширина строки 100
- Относительные импорты (не package-импорты)
- `const`-конструкторы везде, где возможно
- `final` для локальных переменных и полей
- Файлы: `snake_case`, классы: `PascalCase`, константы: `lowerCamelCase`
- Все виджеты должны иметь параметр `key`
- Использовать `for` вместо `.forEach()`
- Не использовать `print()` — использовать логгер Talker
- Не использовать `dynamic`-вызовы

## Окружение

- URL API настраивается через файл `.env` (см. `.env.example`)
- Конфигурация окружения загружается через `flutter_dotenv` и оборачивается в `EnvConfig` (Freezed)
