# Создание ветки и первого коммита

Пользователь передал название реквеста: `$ARGUMENTS`

## Правила преобразования

1. **Название ветки:** взять `$ARGUMENTS`, привести к формату `snake_case`:
   - Заменить ` - ` (пробел-дефис-пробел) на `_`
   - Заменить все оставшиеся пробелы на `_`
   - Привести все буквы к нижнему регистру
   - Пример: `5 - Claude setup` → `5_claude_setup`

2. **Сообщение коммита:** использовать `$ARGUMENTS` как есть.
   - Пример: `5 - Claude setup`

## Шаги выполнения

1. Проверь `git status`.
   - Если есть unstaged изменения (не в staged) — сообщи пользователю и останови выполнение.
   - Staged изменения допустимы — они войдут в первый коммит.
2. Создай новую ветку от текущей: `git checkout -b <имя_ветки>`
3. Создай коммит с названием реквеста: `git commit -m "<название реквеста>"`.
   - Если есть staged изменения — они войдут в этот коммит.
   - Если staged изменений нет — создай пустой коммит: `git commit --allow-empty -m "<название реквеста>"`
4. Сообщи пользователю результат: название ветки и коммит.
