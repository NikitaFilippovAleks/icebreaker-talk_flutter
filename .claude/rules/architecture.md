# Архитектура

> **Правило самообновления:** Обновляйте этот файл при изменении архитектуры, слоёв, настройки DI, маршрутизации или паттернов потока данных.

## Слои

### Слой представления (`lib/presentation/`)
- Организация по фичам: каждая фича содержит `screens/` и опционально `widgets/`
- Каждый экран состоит из небольших виджет-файлов (`index.dart`, `body.dart`, `header.dart` и т.д.)
- Состояние управляется через паттерн **BLoC**: `controller/bloc.dart`, `event.dart`, `state.dart`
- `AppBloc` (`presentation/app/controller/`) — глобальное состояние приложения (загрузка коллекций, инициализация)
- BLoC фич — привязаны к конкретным экранам (например, `CollectionBloc`)

### Слой данных (`lib/data/`)
- **Модели** (`data/models/`) — иммутабельные классы Freezed с JSON-сериализацией
  - Определения таблиц Drift находятся рядом с моделями (например, `collections_table.dart`)
- **Репозитории** (`data/repositories/`) — оркестрируют API и локальные сервисы, реализуют стратегию кэширования
- **Сервисы** (`data/services/`)
  - `api/` — HTTP-клиенты, сгенерированные Retrofit
  - `local/` — объекты доступа к данным Drift

### Инфраструктура (`lib/app/`)
- **DI** (`app/di/`) — GetIt + Injectable. `injector_configurator.dart` инициализирует контейнер; `injector_module.dart` предоставляет Dio, Database и т.д.
- **Роутер** (`app/router/`) — GoRouter с типизированными маршрутами через `go_router_builder`. Маршруты в `routes/index.dart`
- **Тема** (`app/theme/`) — расширения, сгенерированные ThemeTailor. Цвета в `colors.dart`, текстовые стили в `text.dart`
- **База данных** (`app/db/`) — определение базы данных Drift с таблицами
- **Окружение** (`app/env/`) — Freezed-класс `EnvConfig`, оборачивающий значения dotenv
- **i18n** (`app/i18n/`) — JSON-файлы Slang (`en.i18n.json`, `ru.i18n.json`)
- **Сгенерированные ассеты** (`app/gen/`) — вывод flutter_gen для типобезопасного доступа к ассетам

### Общее (`lib/shared/`)
- `extensions/` — Dart-расширения, используемые в разных фичах
- `widgets/` — переиспользуемые виджеты, не привязанные к конкретной фиче

## Поток данных

1. UI отправляет **события BLoC**
2. BLoC вызывает методы **репозитория**
3. Репозиторий координирует **API-сервис** (Retrofit) и **локальный сервис** (Drift)
4. Кэширование: ответы API сохраняются в БД Drift; локальные данные отдаются, когда API недоступен
5. Репозиторий возвращает **модели Freezed** в BLoC
6. BLoC эмитит новое **состояние**, UI перестраивается

## Внедрение зависимостей

- `@singleton` / `@lazySingleton` — сервисы уровня приложения (репозитории, Dio, Database)
- `@injectable` — транзиентные экземпляры (BLoC создаются на каждый экран)
- `@module` — конфигурация сторонних объектов (экземпляр Dio с интерсепторами)
- Доступ через `getIt<T>()` из `injector_configurator.dart`

## Паттерн API-ответов

Все ответы API оборачиваются в `BaseResponse<T>` (`app/api/base_response.dart`) с generic-фабрикой для десериализации.

## Навигация

- GoRouter с shell-маршрутами
- Типизированные data-классы маршрутов с аннотацией `@TypedGoRoute`
- Параметры передаются через конструкторы route data
